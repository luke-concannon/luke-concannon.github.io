'Control Flow' and 'Loops' in Real Life


Ok, Control Flow...


Imagine a book. You know, that dead-tree version of pixels we used to store amazing things in. Your average-joe of a book you read from start to finish. Page 1 - Page [last]. If you've written a bunch of JS code and this experience of reading a book from start to finish is the most relevant metaphor, your JS code is almost unbelievably simple and potentially not viable...


Instead, Control Flow is more like reading a Create Your Own Adventure book. No-way is this read going to be start to finish! You get to a point and there are decisions to be made, dice to be rolled, serious quantum physics to be resolved before the next step can be taken...


This is where loops can really take a hit on your desire to progress in a linear fashion!


A loop in Javascript is like this... You're cruising through your create-your-own-adventure book and you come up against a fork in the road. The book asks "of the 5 characters you are on this adventure with, if more than 3 of them are scared of the dark, take the hill route; if less than 3 are scared of the dark you can take the gloomy valley path (it's quicker, but scarier!")


Another great comparison - Monopoly! Monopoly is control flow and loops in action. Rare is it that you get to cruise though and collect your $200. More common that you go to prison, or get a community chest card that forwards you to Pall Mall. Someone might land on your property and you have to figure out rents for each player (a loop!)


The DOM - What and Why?


HTML, CSS and JS are lietrally different languages. If you imagine them as people speaking vastly different languages, they find it difficult to tell one another what to do. This is where the DOM or Document Object Model comes in handy.


The DOM is a translation of your HTML into a common language that everyone can understand. From here, CSS and JS can use the common language to guide HTML.


Technically speaking, the DOM represents your web content as a hierachical tree of objects, one parent to each child object. JS can identify these objects to affect function, style and content.


Arrays and Objects |The Many Ways to Store Information


Arrays and Objects are two ways we store information in Javascript.


ARRAYS are a way for us to store a list of data in a single variable. They are particularly handy when you want to create ordered collections. Each entry within the array has a number repsenting its position, the first entry sitting as [0].


OBJECTS are a little different - they tend to be things that have information relating to the thing i.e. a person that has a bunch of data relating to who they are (Luke - Age:35, Eye color: blue). An OBJECT stores its information in KEY:VALUE pairs. In the previous example, Age would have been the Key and 35 would have been the value for that key. In this way we access the data in different ways.


The Awesomeness of Functions


In a muddle of confusingly named concepts, FUNCTIONS in JavaScript are probably the most helpfully named concept in the entirety of the computing language. They do what they say on the tin. They make things happen. They take a set of rules and follow those rules. You can pump whatever recipe you want into that function and it will the recipe to the book! Functions are really what makes JS magical, they take information and ask the question "how can we use this information to create something new?"
