What are the differences between absolute, relative and fixed positioning?

TLDR

  • Static - this is the default value, all elements are in order as they appear in the document.
  • Relative - the element is positioned relative to its normal position.
  • Absolute - the element is positioned absolutely to its first positioned parent.
  • Fixed - the element is positioned related to the browser window.
  • Sticky - the element is positioned based on the user's scroll position.

There are 5 main values of the position property.


position: static | relative | absolute | fixed | sticky


What happens when I position: absolute?


An absolutely positioned element is removed from the document flow and is positioned automatically to the top-left of its parent element. It no longer is positioned relative to the overall document, but only to its parent element. By removing it from the webpage flow, you are influencing the position of other elements.


If you imagine a kid is having heaps of fun in a group of kids, they are all collaborating, playing around with each other, then a helicpter parent comes and lifts the kid from the group to play on there own - this is absolute positioning!


What happens when I position: relative?


A relatively positioned element is located relative to itself. It is the badboy of positioning properties, it only cares about itself. "Did you say move 10px left? ok tough guy, but I'm only doing it relative to my original position!"


Relative positioning is such a selfish beast that it does not affect the position of other elements! How rude.


What happens when I position: fixed?


Like position: absolute, fixed positioned elements are also removed from the normal document flow. The differences are:

  • They are only relative to the document, not any other parents.
  • They are not affected by scrolling.
